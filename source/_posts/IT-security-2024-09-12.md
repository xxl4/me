**PyPI Loophole Exposes Numerous Packages to Compromise**

The Python Package Index (PyPI), the official repository for Python packages, has been plagued by a loophole that could allow attackers to compromise thousands of packages. This vulnerability has raised concerns within the Python community.

**Vulnerability Details:**

The vulnerability stems from a loophole in the way PyPI handles metadata. Attackers could exploit this loophole by registering packages with the same name as existing packages but different versions. This would allow them to overwrite the metadata for the legitimate packages, potentially redirecting users to malicious code or stealing sensitive information.

**Affected Packages:**

The impact of this vulnerability is significant, as it affects a vast number of packages. Security researchers have identified over 7,000 packages that are potentially compromised. Some of the notable affected packages include:

* Django
* Flask
* requests
* NumPy
* Pandas

**Mitigation Measures:**

The PyPI team has acknowledged the vulnerability and is actively working on a fix. In the meantime, users are advised to take the following steps to mitigate the risk:

* Update the pip package manager to the latest version (22.1.2 or later)
* Use the `--no-index` flag when installing packages to avoid fetching metadata from PyPI
* Verify the integrity of installed packages using tools like `pip-audit`
* Consider using alternative package managers like conda or Poetry

**Impact on the Python Community:**

The PyPI loophole has shaken the confidence of the Python community. It has highlighted the importance of maintaining the integrity of the package ecosystem and ensuring that users can trust the packages they install. The security community is calling for PyPI to implement stricter measures to prevent such vulnerabilities in the future.

**Conclusion:**

The PyPI loophole is a serious security issue that has compromised the integrity of numerous Python packages. Users are strongly advised to take appropriate precautions to mitigate the risk. The Python community is working together to resolve the vulnerability and restore confidence in the package ecosystem.

## MEF association claims SASE milestone
[Read more](https://www.computerweekly.com/news/366609473/MEF-association-claims-SASE-milestone)

Published: Fri, 30 Aug 2024 11:00:00 GMT

**MEF Association Claims SASE Milestone**

The MEF Association (MEF) has announced a significant milestone in the development of its MEF 3.0 Framework, with the availability of a new set of MEF 3.0 specifications for Secure Access Service Edge (SASE).

**What is MEF 3.0?**

MEF 3.0 is a comprehensive framework for delivering Carrier Ethernet and related networking services. It provides a common language and set of standards for service providers, network operators, and equipment vendors, enabling them to automate and orchestrate their networks.

**What is SASE?**

SASE is a cloud-delivered networking and security solution that combines SD-WAN, firewall, cloud access security broker (CASB), zero-trust network access (ZTNA), and other networking and security capabilities. It aims to provide secure and agile access to applications and data for users located anywhere.

**MEF 3.0 SASE Specifications**

The new MEF 3.0 SASE specifications define the following:

* **Service definition and attributes:** Provides a common definition of SASE services, including performance requirements, security capabilities, and management features.
* **Service ordering and activation:** Establishes a standardized process for ordering, provision, and activation of SASE services.
* **Service assurance and testing:** Specifies test procedures and performance metrics for verifying the quality and reliability of SASE services.
* **Interoperability and vendor certification:** Defines requirements for interoperability between different SASE vendors and provides a certification program to ensure compliance.

**Benefits of MEF 3.0 SASE**

The MEF 3.0 SASE specifications offer several benefits, including:

* **Reduced complexity and cost:** By standardizing SASE services, providers can simplify procurement, deployment, and management, leading to cost savings.
* **Improved security and compliance:** The specifications include stringent security requirements, ensuring that SASE services meet the highest levels of data protection.
* **Increased agility and innovation:** By providing a common framework, MEF 3.0 enables providers to rapidly innovate and launch new SASE services.
* **Enhanced customer experience:** Standardized specifications ensure consistent service levels and performance, leading to a better user experience.

**Industry Reaction**

The MEF 3.0 SASE specifications have been welcomed by the industry. According to Nan Chen, Principal Analyst at Gartner, "The availability of MEF 3.0 SASE specifications addresses a critical need in the market for standardized, interoperable, and certifiable SASE services."

**Conclusion**

The MEF Association's release of MEF 3.0 SASE specifications represents a significant step forward in the development of SASE services. By providing a common framework for definition, ordering, testing, and certification, these specifications will help drive adoption, improve interoperability, and enhance the overall quality and performance of SASE services.

## Iranian APT caught acting as access broker for ransomware crews
[Read more](https://www.computerweekly.com/news/366609258/Iranian-APT-caught-acting-as-access-broker-for-ransomware-crews)

Published: Thu, 29 Aug 2024 10:36:00 GMT

**Iranian APT Acting as Access Broker for Ransomware Crews**

An Iranian state-sponsored advanced persistent threat (APT) group, known as APT35 (also known as Charming Kitten), has been identified acting as an access broker for ransomware gangs. According to a recent report by Mandiant, APT35 has been selling stolen corporate network access to multiple ransomware groups.

**Modus Operandi**

APT35 targets organizations worldwide, primarily in the United States, Europe, and the Middle East. It uses a variety of techniques to gain initial access, including:

* Phishing emails with malicious attachments or links
* Exploiting software vulnerabilities
* Targeting managed service providers (MSPs) to gain access to multiple client networks

Once access is established, APT35 typically moves laterally within the network, stealing credentials and compromising systems. It then exfiltrates sensitive data and sells access to the network to ransomware groups.

**Collaboration with Ransomware Crews**

APT35 has established partnerships with several ransomware gangs, including:

* Hive
* BlackCat
* Conti
* Play

These groups pay APT35 to provide them with access to corporate networks, allowing them to deploy ransomware and encrypt data without having to develop their own infrastructure or target networks independently.

**Impact on Victims**

The collaboration between APT35 and ransomware crews poses a significant threat to organizations. By combining their capabilities, these groups can target and extort victims with increased efficiency and impact.

* **Increased Network Compromises:** APT35's access brokering services enable ransomware groups to bypass traditional network defenses and gain access to otherwise secure networks.
* **Enhanced Ransomware Payouts:** With access to pre-exploited networks, ransomware gangs can demand higher ransoms, as they have a greater chance of successfully encrypting critical data.
* **Data Exfiltration and Extortion:** In addition to encrypting data, APT35 and ransomware crews may also exfiltrate sensitive information from compromised networks, which they use for further extortion attempts.

**Mitigation Measures**

To mitigate the threat posed by APT35 and ransomware crews, organizations should:

* Enhance phishing awareness and employee training
* Patch software vulnerabilities promptly
* Monitor networks for unusual activity and suspicious connections
* Implement multi-factor authentication (MFA)
* Consider using a managed detection and response (MDR) provider
* Back up critical data regularly and test recovery procedures
